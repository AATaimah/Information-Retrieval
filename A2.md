## Goal
The goal of Assignment 2 is to improve the Assignment 1 IR system by applying neural language models (transformer-based) to achieve higher evaluation scores. We use the same SciFact document collection, the same test queries (odd query IDs), and the same evaluation metrics (MAP and P@10). Our approach follows a standard neural IR pipeline: use the classical IR system from Assignment 1 to retrieve up to 100 candidate documents per query, then re-rank these candidates using neural similarity models.

## Approach Overview
1. **Candidate Generation (Assignment 1 baseline)**  
   We use our TF-IDF + cosine similarity retrieval system to retrieve the top 100 candidate documents for each query using the inverted index.

2. **Neural Re-ranking (Assignment 2)**  
   We apply two neural methods to re-score and re-rank the top 100 candidates per query:
   - a **bi-encoder** sentence embedding model (fast)
   - a **cross-encoder** transformer re-ranker (slower but typically stronger)

## Neural Methods Implemented (2 Models)

### Method 1 — Bi-encoder (SentenceTransformer)
- **Model:** `sentence-transformers/all-MiniLM-L6-v2`
- **How it works:** The query and each candidate document are encoded independently into dense embeddings. Relevance is computed via cosine similarity, then candidates are re-ranked by similarity score.
- **Output file:** `Results_minilm`

### Method 2 — Cross-encoder (Transformer Pairwise Re-ranker)
- **Model:** `cross-encoder/ms-marco-MiniLM-L6-v2`
- **How it works:** The model scores each *(query, document)* pair directly, producing a relevance score per candidate. Candidates are re-ranked by this score.
- **Output file:** `Results_cross`

## Files Produced
- `Results_minilm` — bi-encoder reranked run  
- `Results_cross` — cross-encoder reranked run  
- `Results` — best run submitted (currently the cross-encoder run)

## Functionality, Algorithms, Data Structures, and Optimizations

### Candidate Generation
- **Functionality:** reuse the Assignment 1 TF-IDF inverted index to retrieve up to 100 candidate documents per query before neural reranking.
- **Algorithm:** preprocess each query, compute TF-IDF query weights, score documents with cosine similarity, and keep the top 100 results.
- **Data structures:** `Counter` for query term frequencies, `defaultdict(float)` for score accumulation, and the inverted-index structures loaded from JSON.

### Method 1 — Bi-encoder
- **Functionality:** encode the query and candidate documents independently and rerank by cosine similarity in embedding space.
- **Algorithm:** encode the query once, encode candidate documents in batches, compute the dot product on normalized embeddings, then sort in descending score order.
- **Data structures:** candidate document IDs in lists, `doc_id -> {"title", "text"}` stored in a dictionary, and per-query ranking lists of `(doc_id, score)` tuples.

### Method 2 — Cross-encoder
- **Functionality:** score each `(query, document)` pair directly with a transformer reranker.
- **Algorithm:** build all query-document pairs for the top 100 TF-IDF candidates, predict pairwise relevance scores, and rerank candidates by descending score.
- **Data structures:** pair lists `[[query, doc_text], ...]` for model input and sorted `(doc_id, score)` lists for final rankings.

### Optimizations
- Restrict neural scoring to the top 100 TF-IDF candidates instead of the full collection.
- Load each neural model only once and reuse it across all queries.
- Batch bi-encoder and cross-encoder inference with `batch_size=16`.
- Truncate document text to the first 1500 characters before neural scoring to reduce runtime and memory cost.
- Reuse the prebuilt Assignment 1 index rather than rebuilding document representations from scratch.

## Evaluation Results (MAP and P@10)
We evaluated both neural methods using `trec_eval` on the SciFact test qrels.

- **Bi-encoder (all-MiniLM-L6-v2):** MAP = **0.5914**, P@10 = **0.0954**
- **Cross-encoder (ms-marco-MiniLM-L6-v2):** MAP = **0.6345**, P@10 = **0.0922** *(Best MAP)*

**Best run:** Cross-encoder (`Results_cross`), copied as `Results` for submission.

## Comparison with Assignment 1
- Assignment 1 best MAP (`title_plus_text` TF-IDF): **0.4116**
- Assignment 2 bi-encoder MAP: **0.5914**
- Assignment 2 cross-encoder MAP: **0.6345**

Assignment 2 improves substantially over the Assignment 1 baseline. The best MAP rises from **0.4116** to **0.6345**. The cross-encoder performs best because it scores each query-document pair jointly and can capture richer relevance signals than either TF-IDF alone or a bi-encoder. The main tradeoff is efficiency: the cross-encoder is slower and more computationally expensive than the bi-encoder and the classical baseline.
